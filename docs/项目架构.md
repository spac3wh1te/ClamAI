## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 桌面壳 | Tauri 1.x (Rust) | 嵌入 WebView2，提供 IPC |
| 前端 | React 18 + TypeScript + Vite | SPA 单页应用 |
| UI 库 | Tailwind CSS + Radix UI | 样式与组件 |
| 后端 | Go 1.22+ | HTTP 代理服务 |
| 路由 | gorilla/mux | HTTP 路由 |
| 数据库 | SQLite (默认) / PostgreSQL | 数据持久化 |
| 认证 | JWT (HS256) + bcrypt | 管理员/用户认证 |
| 安全 | AC 自动机 + LLM 语义分类 | 内容安全检测 |

## 双端口架构（Provider-native 透明代理）

```

│   Port A (代理端口, 默认 38080)                               │
│   ┌──────────────────────────────────────────────────────┐  │
│   │  Provider-native 透明代理                             │  │
│   │                                                      │  │
│   │  /{provider}/v1/*  → 透传到各服务商原生 API            │  │
│   │  /openai/v1/*       → api.openai.com/v1/*            │  │

│   │                                                      │  │

│   │  - providerMatchMiddleware: 匹配路径 → 注入路由规格    │  │
│   │  - handleTransparentProxy: 透传请求/响应，仅替换认证头 │  │

```

## 透明代理工作原理

```
外部请求                              ClamAI 处理                         上游 API
─────────                            ──────────                         ─────────

POST /siliconflow/v1/chat/completions
  Auth: Bearer clam-sk-xxxx
        │
        ▼
  ① providerMatchMiddleware
     匹配路径前缀 "/siliconflow/v1"
     查找 DefaultProviderRoutes 中的规格
     注入 ProviderRouteSpec 到 context
        │
        ▼
  ② authMiddleware
     验证 clam-sk-xxxx → 数据库 API Key
        │
        ▼
  ③ securityMiddleware
     按 spec.Security 提取请求文本
     关键词/语义安全检测
        │
        ▼
  ④ handleTransparentProxy
     路径重写: /siliconflow/v1/chat/completions
            → /v1/chat/completions
     拼接上游: https://api.siliconflow.cn/v1/chat/completions
     替换认证: Bearer <真实上游 API Key>
     透传请求体/头 → 透传响应
        │
        ▼                                    POST /v1/chat/completions
  ⑤ requestTrackingMiddleware                  Auth: Bearer sk-real-xxxx
     按 spec.Usage 提取 token 用量             Body: {原样透传}
     记录到 request_logs 表          ────→    ──────────────────────
                                           ←── OpenAI 兼容响应（原样返回）
```

## ProviderRouteSpec 路由规格

```go
type ProviderRouteSpec struct {
    Name            string             // "siliconflow"
    PathPrefix      string             // "/siliconflow/v1"
    UpstreamBase    string             // "https://api.siliconflow.cn"
    UpstreamRewrite string             // "/v1"
    AuthType        string             // "bearer" 或 "x-api-key"
    Security        SecurityExtraction // 安全审核字段提取规则
    Usage           UsageExtraction    // 用量统计字段提取规则
}
```

所有 14 个 provider 的路由规格集中在 `DefaultProviderRoutes` 变量中，设计上可扩展为数据库/管理界面配置。

## Provider 接口

```go
type Provider interface {
    GetName() string
    GetBaseURL() string
    GetModels() []string
    GetAPIKey() string
    FetchModels()              // 从上游拉取模型列表
    TestConnection() error     // 测试连通性
}
```

> `ProxyRequest` 方法已弃用，透明代理由 `handleTransparentProxy` 统一处理。

## 认证体系

```
┌──────────────────────────────────────────────────────────────┐
│                        认证流程                               │
│                                                              │
│  管理端口 (/api/v1/*):                                       │
│    1. adminAuthMiddleware 校验 JWT                           │
│    2. PC 模式 (host=127.0.0.1): localhost 请求免认证         │
│    3. Server 模式 (host=0.0.0.0): 所有请求必须带有效 JWT     │
│    4. /auth/status, /auth/login, /auth/setup 等路径免认证     │
│                                                              │
│  代理端口 (/{provider}/v1/*):                                │
│    1. authMiddleware 校验 ClamAI API Key                     │
│    2. 支持: JWT token / 静态 API Key (clam-sk-xxx)          │
│    3. X-Internal-Analysis 头: 内部调用免认证                 │
│    4. 验证通过后，handleTransparentProxy 替换为上游真实 Key   │
│                                                              │
│  Token 生命周期:                                             │
│    Access Token:  2 小时有效                                 │
│    Refresh Token: 30 天有效, 一次性使用                       │
│    JWT Secret:    存储在 DB admin_secrets 表, 持久化         │
└──────────────────────────────────────────────────────────────┘
```

## 数据库 Schema (核心表)

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `users` | 用户账户 | id, username, password_hash(bcrypt), role(admin/user) |
| `providers` | AI 提供商 | id, name, provider_type, base_url, api_keys(JSON), models(JSON) |
| `api_keys` | 网关密钥 | id, key(clam-sk-xxx), user_id, allowed_models(JSON), active |
| `request_logs` | 请求日志 | timestamp, provider, model, tokens, latency, api_key_used |
| `security_config` | 安全策略 | config_json (完整 SecurityConfig 序列化) |
| `security_alerts` | 安全告警 | timestamp, direction, trigger_type, trigger_detail |
| `rate_limit_config` | 限流规则 | config_json |
| `analysis_tasks` | 分析任务 | id, task_no, model, schedule_type, status, result |
| `skills_tasks` | 技能检测任务 | id, task_no, name, model, source_type, source_info, status, result |
| `skills_task_history` | 技能检测历史 | task_id, status, result_risk_level, started_at, completed_at |

## 构建模式

```bash
# PC 模式 (桌面应用)
build.bat x86_64-win        # 生成 ClamAI.exe + ClamAI-service.exe

# Web/Server 模式 (独立服务)
go build -tags server       # 编译带嵌入式前端的 ClamAI-service.exe

# PC 模式 Go 服务 (无前端)
go build                    # 普通 Go 编译
```
