## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 桌面壳 | Tauri 1.x (Rust) | 嵌入 WebView2，提供 IPC |
| 前端 | React 18 + TypeScript + Vite | SPA 单页应用 |
| UI 库 | Tailwind CSS + Radix UI | 样式与组件 |
| 后端 | Go 1.22+ | HTTP 代理服务 |
| 路由 | gorilla/mux | HTTP 路由 |
| 数据库 | SQLite (默认) / PostgreSQL | 数据持久化 |
| 认证 | JWT (HS256) + bcrypt | 管理员/用户认证 |
| 安全 | AC 自动机 + LLM 语义分类 + 正则威胁规则 + 系统行为分析 | 内容安全检测 |

## 认证体系

```
┌──────────────────────────────────────────────────────────────┐
│                        认证流程                               │
│                                                              │
│  管理端口 (/api/v1/*):                                       │
│    1. adminAuthMiddleware 校验 JWT                           │
│    2. PC 模式 (host=127.0.0.1): localhost 请求免认证         │
│    3. Server 模式 (host=0.0.0.0): 所有请求必须带有效 JWT     │
│    4. /auth/status, /auth/login, /auth/setup 等路径免认证     │
│                                                              │
│  代理端口 (/{provider}/v1/*):                                │
│    1. authMiddleware 校验 ClamAI API Key                     │
│    2. 支持: JWT token / 静态 API Key (clam-sk-xxx)           │
│    3. X-Internal-Analysis 头: 内部调用免认证                  │
│    4. 验证通过后，handleTransparentProxy 替换为上游真实 Key    │
│                                                              │
│  Token 生命周期:                                             │
│    Access Token:  2 小时有效                                 │
│    Refresh Token: 30 天有效, 一次性使用                       │
│    JWT Secret:    存储在 DB admin_secrets 表, 持久化         │
└──────────────────────────────────────────────────────────────┘
```

## 数据库 Schema (核心表)

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `users` | 用户账户 | id, username, password_hash(bcrypt), role(admin/user) |
| `providers` | AI 提供商 | id, name, provider_type, base_url, api_key, models(JSON), disabled_models(JSON), priority |
| `api_keys` | 网关密钥 | id, key(clam-sk-xxx), user_id, allowed_models(JSON), active |
| `request_logs` | 请求日志 | timestamp, provider, model, tokens, latency, api_key_used(掩码), is_proxy_call, upstream_* |
| `security_config` | 安全策略 | config_json (完整 SecurityConfig 序列化) |
| `security_alerts` | 安全告警 | timestamp, direction, trigger_type, trigger_detail, severity, content_preview, api_key_used(掩码) |
| `threat_rules` | 威胁规则 | threat_type, name, patterns_json, severity, enabled |
| `rate_limit_config` | 限流规则 | config_json |
| `analysis_tasks` | 用户分析任务 | id, task_no, model, schedule_type, status, result, created_by |
| `skills_tasks` | 技能检测任务 | id, task_no, name, model, source_type, source_info, status, result, created_by |
| `skills_task_history` | 技能检测历史 | task_id, status, result_risk_level, started_at, completed_at |
| `system_analysis_config` | 系统行为分析配置 | enabled, model, api_key_id, time_range, interval_minutes, notify_on_high_risk |
| `system_analysis_tasks` | 系统分析任务 | id, task_no, name, api_key_id, model, schedule_type=periodic, status, result_*（created_by='__system__'） |
| `system_analysis_task_history` | 系统分析历史 | task_id, risk_level, summary, detail, dimensions, logs_analyzed, run_at |

## 构建模式


## 已知问题修复记录

### Provider API Key 掩码问题（已修复）

**问题**：切换主题时，Provider 的 API Key 被覆写成缩略值（如 `sk-s...eknd`）。

**根因**：`handleGetConfig` 在返回配置时对 `providers.ApiKeys[].KeyValue` 做了掩码处理。前端切换主题时通过 `GET /config` → `PUT /config` 保存完整配置（包含已掩码的 key），导致数据库被覆写。

**修复**：从 `handleGetConfig` 中移除对 Provider API Keys 的掩码处理（保留对 Gateway API Key 的掩码）。`/api/v1/providers` 和 `/api/v1/config` 现均返回完整 key 值。

**影响范围**：此修复之前的主题切换操作会导致 provider key 被掩码覆盖，重新保存即可恢复。
