# ClamAI 快速使用指南

ClamAI 是一个 AI 安全护栏平台，采用 **Provider-native 透明代理架构**，按服务商原生协议透传转发，支持多提供商管理、安全防护、威胁检测、系统行为分析、AI 智能体安全、用量统计等功能。可部署为服务器网关或终端 EDR。

## 三种部署模式

| 模式 | 适用场景 | 架构 |
|------|----------|------|
| **PC 本地模式** | 个人开发者在本地电脑使用 | Tauri 桌面应用 + 本地 Go 服务 |
| **Web 服务模式** | 团队/组织部署在服务器上 | Go 服务 + 嵌入式 Web UI |
| **远程连接模式** | 连接到已部署的远程服务器 | Tauri 桌面应用 → 远程 Go 服务 |

## 一、PC 本地模式（推荐首次使用）

### 前置条件
- Windows x86_64 或 ARM64
- 无需安装额外运行时

### 首次运行步骤

#### 1. 启动应用
双击 `ClamAI.exe` 启动桌面应用。

首次启动会进入**设置向导**，分为三步：

#### 2. 第一步：选择模式
选择「PC 本地模式」（默认）。应用会自动检测可用端口。

#### 3. 第二步：配置服务
- 点击「检测端口」确认端口可用（默认代理端口 38080，管理端口 38081）
- 点击「启动服务」等待服务就绪

#### 4. 第三步：创建管理员
- 设置管理员用户名和密码（密码至少 6 位）
- 点击「完成」

#### 5. 添加 AI 提供商
进入管理界面后：
1. 左侧导航选择「模型管理」
2. 点击「添加提供商」
3. 选择提供商类型（如 SiliconFlow）或自定义
4. 填入 Base URL 和 API Key
5. 点击「测试连接」验证可用性
6. 点击「获取模型」拉取可用模型列表
7. 保存

#### 6. 创建 API Key
1. 左侧导航选择「模型管理 → API密钥」Tab
2. 点击「创建密钥」
3. 设置名称，选择允许的模型
4. 复制生成的密钥（格式 `clam-sk-xxxx`），**仅显示一次**

#### 7. 在外部工具中使用

ClamAI 采用 **Provider-native 透明代理**架构，外部工具按服务商选择对应的路径前缀：

| 服务商 | API 地址 | 说明 |
|--------|----------|------|
| SiliconFlow | `http://localhost:38080/siliconflow/v1` | OpenAI 兼容格式 |
| DeepSeek | `http://localhost:38080/deepseek/v1` | OpenAI 兼容格式 |
| OpenAI | `http://localhost:38080/openai/v1` | OpenAI 原生格式 |
| Anthropic | `http://localhost:38080/anthropic/v1` | Anthropic 原生格式 |
| 智谱 GLM | `http://localhost:38080/glm/v1` | GLM 原生格式 |
| 通义千问 | `http://localhost:38080/qwen/v1` | OpenAI 兼容格式 |
| 豆包 | `http://localhost:38080/doubao/v1` | 字节跳动格式 |
| Moonshot | `http://localhost:38080/moonshot/v1` | OpenAI 兼容格式 |
| Yi | `http://localhost:38080/yi/v1` | OpenAI 兼容格式 |
| OpenRouter | `http://localhost:38080/openrouter/v1` | OpenAI 兼容格式 |

**API Key**：上一步创建的 `clam-sk-xxxx`

> **关键变化**：ClamAI 不再做协议转换，而是按服务商原生协议透传。工具配置的 API 地址需带 provider 前缀（如 `/siliconflow/v1`），ClamAI 会自动将请求透传到对应的上游 API。

### CherryStudio 配置示例

1. 添加自定义服务商
2. API 地址：`http://localhost:38080/siliconflow/v1`
3. API Key：`clam-sk-xxxx`
4. 点击「获取模型列表」→ 自动从 `GET /v1/models` 获取所有可用模型

### Claude Code 配置示例

```bash
# 设置环境变量
export ANTHROPIC_BASE_URL=http://localhost:38080/anthropic/v1
export ANTHROPIC_API_KEY=clam-sk-xxxx
```

## 二、Web 服务模式

### 部署方式

使用带 `server` 构建标签编译的 `ClamAI-Server.exe`，它内嵌了前端静态资源。

#### 1. 编译

```bash
# 先构建前端
cd src
npm install && npm run build

# 将前端产物复制到 Go 可访问目录
cp -r dist/ ../server/frontend/dist/

cd ../server
go build -tags server -o ClamAI-Server.exe .
```

或使用项目构建脚本：
```bash
build.bat x86_64-win
```

#### 2. 启动

```bash
ClamAI-Server.exe --host 0.0.0.0 --port 8080 --admin-port 8081
```

参数说明：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--host` | `127.0.0.1` | 监听地址，`0.0.0.0` 表示所有接口 |
| `--port` | `8080` | 代理服务端口（对外提供模型 API） |
| `--admin-port` | `8081` | 管理端口（Web UI + 管理 API） |
| `--ssl` | false | 启用 HTTPS |
| `--log-level` | `info` | 日志级别 |

#### 3. 首次初始化

浏览器访问 `http://服务器IP:8081/admin/`，首次访问进入设置向导创建管理员账户。

#### 4. 对外使用

外部工具配置（以 SiliconFlow 为例）：
- **API 地址**：`http://服务器IP:8080/siliconflow/v1`
- **API Key**：管理界面中创建的 `clam-sk-xxxx`

### 多用户
- 管理员可在「管控中心 → 用户管理」中创建普通用户
- 普通用户登录后可管理自己的 API Key
- 支持 API Key 级别的模型访问控制

## 三、远程连接模式

适用于已有团队部署的 ClamAI 服务，个人通过桌面应用连接。

### 步骤

#### 1. 启动桌面应用
双击 `ClamAI.exe`

#### 2. 设置向导
1. 选择「远程连接模式」
2. 填入远程服务地址（如 `http://192.168.1.100:8081`）
3. 点击「测试连接」验证可达性
4. 如果远程服务未初始化，会提示创建管理员
5. 如果已初始化，直接使用已有账户登录

#### 3. 使用
连接成功后，操作方式与 PC 本地模式相同。

## 四、侧边栏导航说明

登录后左侧导航分为以下区块：

```
├── 首页
│   └── 安全总览（仪表盘：请求趋势图、Provider 用量、安全告警、TOP10 排行）
├── 防护中心
│   ├── 实时防护   — 实时安全告警（内容安全拦截事件：关键词/语义/向量）
│   ├── 威胁挖掘   — 系统级 AI 行为分析（KeyResult 扁平表格、威胁评分、维度分析）
│   └── 安全广场   — 安全工具集（调用者行为分析/Skills检测/Agent环境检查）
├── 管控中心
│   ├── 模型管理   — 提供商/模型/ApiKeys 三 Tab 管理
│   ├── 用户管理   — 用户账户 CRUD（管理员可见）
│   ├── 密钥管控   — API Key 管控（管理员可见）
│   └── 流量控制   — 限流规则配置（管理员可见）
├── 审计中心（管理员可见）
│   ├── 模型调用日志 — API 调用日志（高级搜索/筛选/导出）
│   └── 系统运行日志 — 服务端日志（log lines 浏览）
└── 设置中心
    ├── 基础设置   — 主题/语言/代理/服务连接
    └── 安全设置   — 实时防护配置/关键词/语义/向量/威胁分析（管理员可见）
```

> **注意**：主题切换（设置页面、侧边栏主题选择）走 `GET/PUT /config` API，会完整保存/恢复 Provider API Keys，无需担心 key 被覆盖。

## 五、核心功能说明

### 威胁挖掘（系统行为分析）
系统级 AI 行为分析，完全独立于用户分析任务，自动周期性执行（30秒调度）：
- 在「安全中心 → 威胁挖掘」页面查看
- 在「防护策略 → 威胁分析配置」中配置（分析模型、间隔、API Key、时间范围、高风险通知等）
- 扁平表格展示各 API Key 的威胁分析结果（评分、信号、维度、AI 报告）
- system_analysis 结果不再写入 security_alerts，通过 KeyResult 在威胁挖掘页面展示

### 威胁规则
正则表达式驱动的实时威胁检测，覆盖 4 类威胁：黑客攻击、模型越狱、对抗攻击、恶意内容生成。在「防护策略 → 高级策略 → 威胁规则」中管理。

### 安全告警
支持 severity 级别（critical/high/medium/low），可筛选、可搜索、可标记已处理。告警来源分两类：内容安全（关键词/语义/向量拦截）和威胁分析。实时防护页面仅显示内容安全告警。

### 日志自动刷新
日志页面的服务日志自动刷新功能**默认关闭**，避免干扰搜索和筛选操作。需要时可点击刷新按钮开启。

## 常见问题

### Q: 外部工具无法连接代理服务？
**PC 模式**：确认防火墙允许对应端口通信。

**Web 模式**：确认 `--host 0.0.0.0` 且服务器防火墙开放了代理端口。

### Q: 如何选择正确的 API 地址？
ClamAI 按服务商原生协议透传，不同服务商需要使用不同的路径前缀：
- OpenAI 兼容的服务商（SiliconFlow、DeepSeek、Qwen 等）使用 `/{provider}/v1` 作为 API 地址
- Anthropic 原生使用 `/anthropic/v1` 作为 API 地址
- 具体前缀见上方「在外部工具中使用」章节的表格

### Q: 忘记管理员密码？
删除数据库文件 `clamai.db` 重新初始化（会丢失所有配置）。

### Q: 如何切换部署模式？
桌面应用中：设置 → 部署模式 → 切换 → 重启应用。

### Q: Provider API Key 变成了 `sk-s...eknd` 缩略值？
这是已知问题，已在最新版本中修复。请重新打开管理界面，从「模型管理」中重新保存一次 Provider 配置即可恢复正常。如果问题仍然存在，可通过 `PUT /api/v1/providers/{name}/key` 接口重新设置 key。

### Q: 切换主题会导致 Provider API Key 被覆盖吗？
不会。最新版本已修复此问题——主题切换时 `/config` API 返回完整 Provider API Key，不会再做掩码处理。
